Release Process

First time set up

  • Make sure you have at least a Committer role
  • Make sure you have permission to post to the Announcement Blog.
  • Check out the google-api-java-client and javadoc projects as follows:
    cd rootDir
hg clone https://google-api-java-client.googlecode.com/hg/ google-api-java-client
hg clone https://javadoc.google-api-java-client.googlecode.com/hg/ javadoc
  • Download JDiff.
  • Set up for pushing to central Maven repository
    • Sign up for a Sonatype JIRA account at https://issues.sonatype.org. Click Sign Up in the Login box, fill out the simple form, and click Sign Up.
    • Install a GPG client from http://www.gnupg.org/, and then create a GPG key using these instructions, making sure to specify a passphrase to be used every time you release. Make sure to run the instructions under "Distribute Your Public Key".
    • Add these lines to your ~/.hgrc: 
      
[ui]
username = Your Name 

[auth]
google-api-java-client.prefix = https://google-api-java-client.googlecode.com/hg/
google-api-java-client.username = [username from https://code.google.com/hosting/settings]
google-api-java-client.password = [password from https://code.google.com/hosting/settings]
    • Add these lines to your ~/.hgrc: 
      google-api-java-client.prefix = https://google-api-java-client.googlecode.com/hg/
google-api-java-client.username = [username from https://code.google.com/hosting/settings]
google-api-java-client.password = [password from https://code.google.com/hosting/settings]
    • Add these lines to your ~/.m2/settings.xml (create this file if it does not already exist): 
      <settings>
  <servers>
    <server>
      <id>google-snapshots</id>
      <username>[username for https://issues.sonatype.org]</username>
      <password>[password for https://issues.sonatype.org]</password>
    </server>
    <server>
      <id>google-releases</id>
      <username>[username for https://issues.sonatype.org]</username>
      <password>[password for https://issues.sonatype.org]</password>
    </server>
    <server>
      <id>sonatype-nexus-snapshots</id>
      <username>[username for https://issues.sonatype.org]</username>
      <password>[password for https://issues.sonatype.org]</password>
    </server>
    <server>
      <id>sonatype-nexus-staging</id>
      <username>[username for https://issues.sonatype.org]</username>
      <password>[password for https://issues.sonatype.org]</password>
    </server>
  </servers>
</settings>

Release version 1.4.2-beta

  • Prepare the project 
    • cd rootDir/google-api-java-client
hg status
hg out
      to make sure you have no changes that have not yet been committed or pushed to the Mercurial server
  • Push to central Maven repository (based on instructions from Sonatype OSS Maven Repository Usage Guide):
    • Prepare:
      cd rootDir/google-api-java-client
mvn clean deploy
mvn release:clean
    • mvn release:prepare -DautoVersionSubmodules=true
      • Accept default for 'What is the release version for "Google API Client - Parent"?'
      • Type 1.4.2-beta for 'What is SCM release tag or label for "Google API Client - Parent"?'
      • Accept default for 'What is the new development version for "Google API Client - Parent"?'
    • mvn release:perform -Darguments=-Dgpg.passphrase=[YOUR GPG PASSPHRASE]
    • Release on oss.sonatype.org
      • Log in to https://oss.sonatype.org
      • Click on "Staging Repositories" and look for the "com.google" Repository with your username
      • check box next to uploaded release
      • click Close button; type 1.4.2-beta and click Close
      • check box next to uploaded release
      • click Release button; type 1.4.2-beta and click Release
    • Central Maven repository is synced hourly. Once uploaded onto repo2.maven.org, it will be found at: repo2.maven.org
  • Start development of next bug fix version
    • hg update
    • For the following files, replace 1.4.2-beta with the next bug fix version:
      • jdiff.xml
      • release.html
      • google-api-client/src/main/java/com/google/api/client/util/Strings.java
    • hg commit -m "start next bug fix version"
hg push
  • Prepare the released library zip file and javadoc
    • JDiff is a tool for showing the difference between the JavaDoc of one version of the library vs. the JavaDoc of an old version of the library. Hence, we will check out the last minor version of the library into a temporary directory and compare against the JavaDoc of the just released version of the library, and then hg commit that diff into the javadoc Hg repository.
    • change these constants from jdiff.xml for your environment: JDIFF_HOME, MAVEN_REPOSITORY_HOME, JAVADOC_PROJECT_ROOT_DIR
    • cd /tmp
rm -rf google-api-java-client
mkdir google-api-java-client
cd google-api-java-client
hg clone -r 1.3.2-alpha https://google-api-java-client.googlecode.com/hg/ 1.3.2-alpha
hg clone -r 1.4.2-beta https://google-api-java-client.googlecode.com/hg/ 1.4.2-beta
cd 1.4.2-beta
mvn javadoc:jar install site
cp -R target/site/apidocs/ rootDir/javadoc/1.4.2-beta
ant -f jdiff.xml
cd rootDir/javadoc
hg add
hg commit -m "1.4.2-beta"
hg push
cd rootDir/google-api-java-client
  • WAIT! Don't do the following instructions until after the library has been uploaded to repo2.maven.org at: repo2.maven.org
  • Update to new version on http://code.google.com/p/google-api-java-client
    • Upload to http://code.google.com/p/google-api-java-client/downloads/entry
      • Summary: Google API Client Library for Java, version 1.4.2-beta
      • File: /tmp/google-api-java-client/1.4.2-beta/google-api-client-assembly/target/google-api-client-1.4.2-beta-java.zip
      • Labels: Type-Archive, OpSys-All, and Featured
      • click Submit file
    • If it is a bug fix release, deprecate the library from any previous versions by removing any Featured label and adding the Deprecated label.
    • Update the following pages changing any reference from the previous version to the new version 1.4.2-beta:
      • admin
      • wiki/Setup
      • wiki/SampleProgram
  • New Post on the announcement blog about the new version, including links to new features and bugs fixed.
